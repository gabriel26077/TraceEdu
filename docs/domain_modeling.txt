================================================================================
AGREGADOS E ENTIDADES / AGGREGATES AND ENTITIES
================================================================================

0. AGREGADO: ESCOLA / SCHOOL AGGREGATE (TENANT ROOT)
--------------------------------------------------------------------------------
School (Raiz/Root):
    - uid
    - name (Nome da Instituição)
    - representatives: Representative[] (Value Object list)
    - coordination_email: Email (VO)
    - coordination_phone: string
    - settings: SchoolSettings (VO - ex: recovery_policy, term_count)

Representative (Value Object):
    - user_id (Referência ao User)
    - description (Ex: "Diretor Financeiro", "Coordenador Pedagógico")
    - contact: string

1. AGREGADO: USUÁRIO / USER AGGREGATE
--------------------------------------------------------------------------------
User (Raiz/Root):
    - uid
    - school_id (Referência à escola atual)
    - name
    - email: Email (VO)
    - birthdate (opcional)
    - cpf: CPF (VO)
    - roles: ["student", "teacher", "admin", "clerk"]
    - status: "active" | "transferred" | "inactive"

Account (Entidade/Entity):
    - uid
    - user_id
    - username (Global ou por Escola)
    - password_hash
    - status: "active" | "inactive"

2. AGREGADO: MATÉRIA / SUBJECT AGGREGATE
--------------------------------------------------------------------------------
Subject (Raiz/Root):
    - uid
    - school_id (Cada escola define sua grade/curriculo)
    - name
    - description
    - level: "Elementary 1" | "Elementary 2" | "High School"
    - academic_units: int
    - offering_type: "in-person" | "online" | "hybrid"

3. AGREGADO: GRUPO / CLASS GROUP AGGREGATE
--------------------------------------------------------------------------------
ClassGroup (Raiz/Root):
    - uid
    - school_id
    - name
    - shift: "morning" | "afternoon" | "night"
    - student_ids: User[]
    - base_subject_offering_ids: SubjectOffering[]

4. AGREGADO: OFERTA / SUBJECT OFFERING AGGREGATE
--------------------------------------------------------------------------------
SubjectOffering (Raiz/Root):
    - uid
    - school_id
    - subject_id
    - teacher_ids: User[]
    - period: AcademicPeriod (VO - Ex: "2026.1")
    - enrollment_ids: Enrollment[]

5. AGREGADO: DESEMPENHO / ENROLLMENT AGGREGATE
--------------------------------------------------------------------------------
Enrollment (Raiz/Root):
    - uid
    - school_id
    - student_id
    - subject_offering_id
    - total_absences: int
    - status: "enrolled" | "approved" | "failed" | "dropped" | "transferred"
    - grades: Grade[]

Grade (Entidade/Entity):
    - uid
    - term: int
    - value: AcademicGrade (VO)
    - grade_type: "regular" | "recovery" | "final"

================================================================================
OBJETOS DE VALOR / VALUE OBJECTS (VOs)
================================================================================

- CPF: Validação de formato e dígitos. Imutável.
- Email: Validação de formato.
- AcademicGrade: Garante valor entre 0.0 e 10.0.
- AcademicPeriod: Garante formato "YYYY.S".
- SchoolSettings: Configurações específicas da escola (ex: média de aprovação).

================================================================================
NOTAS DE MODELAGEM / MODELING NOTES
================================================================================

1. [PT] Multi-tenancy: O school_id é obrigatório em todas as consultas para isolamento.
   [EN] Multi-tenancy: school_id is mandatory in all queries for data isolation.

2. [PT] Migração de Alunos: Ao transferir um aluno, o status na escola de origem muda para "transferred" e as matrículas ativas são encerradas. Um novo registro é criado na escola de destino.
   [EN] Student Migration: When transferring a student, their status in the origin school changes to "transferred" and active enrollments are closed. A new record is created in the destination school.

3. [PT] Professores Multi-Escola: Um professor pode atuar em várias escolas, mas sua conta é global. O acesso é controlado por sua presença no aggregate School como representante ou teacher.
   [EN] Multi-School Teachers: A teacher can work in multiple schools, but their account is global. Access is controlled by their presence in the School aggregate as representative or teacher.
